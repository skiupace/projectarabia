# Mission & Vision

## What is ProjectArabia?

ProjectArabia is a **tech-focused community platform** for link aggregation and thoughtful discussion, inspired by Hacker News and Lobsters, but built for the modern web and the Arabic tech scene.

## Target Audience

- **Developers** building products and solving problems
- **Founders** sharing their journey and lessons
- **Builders** creating in public
- **Curious minds** who value depth over noise

Not for:
- Engagement farmers
- Spam/promotional content
- Shallow clickbait
- Algorithm-driven hype

## Core Values

### 1. Depth Over Noise
- Quality discussions, not quantity
- Thoughtful comments, not hot takes
- Signal, not noise

### 2. Simplicity
- No complex features for the sake of it
- Predictable UX
- Fast and minimal
- No hidden magic

### 3. Community-First
- User-owned conversations
- Fair moderation
- No engagement tricks
- Respect for time and attention

### 4. Edge-Native
- Fast global access
- Modern architecture
- Scalable by design
- No legacy cruft

## Key Features

### Link Sharing
- Submit tech articles, projects, or discussions
- Clean URL parsing and metadata
- No clickbait optimization

### Voting System
- Simple up/down votes
- Fair ranking algorithm (no gaming)
- Reflects genuine community interest

### Comments
- Threaded discussions
- Real-time updates via SSR
- Reply notifications (non-spammy)

### User Profiles
- Submission history
- Comment history
- Voting activity
- Badge system (earned, not bought)

### Content Discovery
- **Newest**: Latest submissions
- **Ask**: Questions and discussions
- **Past**: Archive by date
- Clean, chronological feeds

## Design Philosophy

### Old-School Web, Modern Tech
- Inspired by HN/Lobsters simplicity
- Built with cutting-edge stack
- Fast, not flashy
- Functional, not fancy

### No Dark Patterns
- No infinite scroll addiction
- No engagement farming
- No algorithmic manipulation
- No hiding information

### Respect User Time
- Fast load times (edge-native)
- No unnecessary animations
- Clear information hierarchy
- Minimal cognitive load

## Technical Philosophy

### Edge-First
Everything runs on Cloudflare's global network:
- **Workers**: Compute everywhere
- **D1**: Data close to users
- **KV**: Instant cache access
- **Queues**: Async processing

### Serverless-First
- No traditional servers
- Pay for what you use
- Infinite scale potential
- Zero maintenance overhead

### Real-Time Consistency
- KV for speed (sessions, cache)
- D1 for truth (persistent data)
- No stale data for users

### Predictable Simplicity
- No ORMs hiding complexity (Drizzle is transparent)
- No magic abstractions
- Clear data flow
- Easy to reason about

- Let votes decide quality
- Civil disagreement OK
